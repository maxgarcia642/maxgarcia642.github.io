   specials: aero · glass · frost · storm · ocean · indigo · lavender · royal ·
   sakura · synth · crimson · coral · amber · desert · coffee · luxe · citrus ·
   matcha · forest · mint · neon · crt · pixel · brutal · vapor · holo · paper ·
   noir · bauhaus · blueprint.
   LEGACY IDs MIGRATE (saved localStorage keeps working): aurora→indigo,
   clay→coral, swiss→bauhaus, midnight→noir — mapped in the FOUC guard AND
   setTheme(). ⚠ The FOUC guard changed → CSP sha256 recomputed
   ('sha256-akJOeUYgicp+AUQ4Nap0oxlZS6dt7c3t1+Pji9ugtFI='). Edit that inline
   script even by a space and you must recompute.
   EVERY theme now declares ~40 tokens including fonts (11-family Google pool),
   shadows, gradients, radii, border weights, gloss, --bg-anim (drifting
   gradient / hue sweep / scanline flicker / scrolling grids / synthwave
   horizon), and --fx + --fx-color for the particle engine. Ambient layers:
   CRT+Amber scanlines, Neon animated grid, Blueprint static grid, Synthwave
   perspective floor, Brutal/Bauhaus hard offset shadows, Pixel
   image-rendering (PDF canvases exempted), Holo iridescent borders.

4. THEME AFFECTS FONTS / SHADOWS / GRADIENTS / PARTICLES → YES, ALL FOUR
   Fonts: per-theme display+body pairs (Playfair for velvet/luxe, Press Start
   for pixel, Caveat for paper, Orbitron for synth/vapor, Lora for coffee…).
   Particles: native FX ENGINE in script.js — 16 presets (bubbles, orbs, snow,
   rain, embers, fireflies, stars, petals, leaves, motes, glyph-rain, confetti,
   shapes, prisms, steam, dust) as CONFIG, not code paths. Reads --fx/--fx-color
   from the active theme. Respects prefers-reduced-motion, the Layout Settings
   motion kill-switch, density setting, and runs at reduced density on mobile
   (backgrounds now live on phones too, at ~55% particle count, DPR-capped).
   Round-2 scout repos (partikle, JParticles, ab-particles, marcbruederlin —
   all MIT-verified) were idea-mined for preset feel; ZERO code copied — house
   rule stays zero-dependency.

5. MUSIC: AUTOPLAY ON ENTRY + RED CROSS MUTE + PLAYLIST + SKIP + UPLOADS
   Honesty first: Chrome/Safari/Firefox BLOCK audible autoplay pre-interaction.
   "Autoplay on entry" is implemented as the closest honest thing: play() is
   attempted immediately on load, and if the browser refuses, the first
   click/keypress anywhere starts it (unless muted last visit). Documented in
   assets/music/README.md and surfaced in Layout Settings copy — no pretending.
   Mute: ♫ button gains a diagonal red cross overlay (CSS ::after) when muted.
   Playlist: content.json → music.tracks manifest; 1 track loops, 2+ advance on
   'ended'; ⏭ dock button appears automatically at 2+. Upload (🎧) accepts any
   browser-decodable audio format, joins the rotation as a local object URL
   (never uploaded), and starts playing. AIR licensing flag: retained verbatim,
   Max's documented accepted risk — not re-litigated, not silently kept either.
